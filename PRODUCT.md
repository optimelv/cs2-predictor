# Product

## Register

product

## Users

CS2 esports analysts, bettors, and model builders who want a fast read on upcoming match probabilities, confidence, tier quality, and model caveats. They use the app before matches, around veto time, and after data refreshes to decide where the model has a usable signal.

## Product Purpose

Expose the CS2 predictor as a static GitHub Pages web app backed by generated JSON data. Success means a user can see the current benchmark quality, inspect upcoming or benchmark match predictions, understand pre-match versus post-veto confidence, and know when the data was last refreshed.

## Brand Personality

Sharp, analytical, honest. The product should feel like a credible esports trading desk: confident enough to guide decisions, careful enough to show limits, and visually distinct from generic sportsbook pages.

## Anti-references

Avoid sportsbook hype, fake certainty, AI-purple gradients, generic glass cards, crypto-dashboard clutter, and claims that post-veto accuracy applies before veto data is known.

## Design Principles

- Show the model's edge and its uncertainty in the same view.
- Separate pre-match predictions from post-veto predictions clearly.
- Treat tier and integrity context as first-class quality signals.
- Make freshness visible; stale data should never look live.
- Keep the interface dense enough for analysis without burying the strongest picks.

## Accessibility & Inclusion

Target WCAG AA. Support keyboard navigation, visible focus states, reduced motion, high text contrast, and do not rely on color alone for prediction status or confidence.
